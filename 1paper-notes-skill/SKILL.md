# 读论文生成笔记

## 目标

基于用户提供的论文（PDF或截图），生成一份精简的结构化笔记（约1-2页），让读者能在3分钟内抓住这篇论文的核心。笔记面向文献分享场景，需要让没读过原文的人也能快速理解。

## 工作流程

### 1. 获取论文内容

- 优先阅读用户提供的PDF文件
- 如果用户只提供标题，通过搜索获取论文信息

### 2. 搜索引用数

通过 Google Scholar 搜索论文标题，获取当前被引次数。在笔记的基本信息中标注。如果搜索失败，标注"引用数待查"而非留空。

### 3. 生成笔记

按照下方模板结构输出。除"方法概述"外，其余模块控制在2-4句话。

### 4. 图表分析

重点分析论文中的实验方法流程图（通常是Figure 1或Method部分的示意图）。逐模块拆解：输入是什么 → 每一步做了什么操作 → 输出是什么。如果用户提供了图片截图，直接基于图片分析；否则基于正文中对方法的描述来还原流程。

### 5. 相关论文推荐

从论文的 References 列表中，挑出3-5篇最相关的文献（优先选择：被正文反复引用的、方法上直接对比的baseline、同一研究线上的前序工作）。给出标题、作者、年份，以及一句话说明为什么值得关注。

## 笔记模板

```markdown
# [论文标题]

## 基本信息

| 项目 | 内容 |
|------|------|
| 作者 | 第一作者 et al. |
| 期刊/会议 | 名称，年份 |
| DOI | xxx |
| 引用数 | Google Scholar 当前被引次数 |

## 核心贡献（一句话）

用一句话概括这篇论文最重要的贡献是什么。

## 研究动机与背景

- 领域中存在什么问题/gap
- 现有方法有什么不足
- 这篇论文打算怎么解决（2-3句话）

## 名词解释

- **术语A（English A）**：一句话解释，在本文中具体指什么
- **术语B（English B）**：一句话解释
- ...（列出5-8个关键术语）

## 方法概述

### 输入数据
方法接收什么数据，格式和维度

### 核心步骤
1. **步骤名称**：做了什么操作 → 为什么这么做 → 输出什么
2. **步骤名称**：...
3. ...（通常3-6步）

### 关键设计选择
- 为什么选择某方案而非替代方案
- 关键超参数及其作用

### 输出
最终输出是什么，如何用于下游分析

## 关键结果

- 主要实验结论（2-3条）
- 和baseline比有什么提升
- 用什么数据集/指标验证的

## 个人思考/评价

> [用户自行填写：对方法的看法、局限性、和自己研究的关联、可能的改进方向]

## 关键图表分析

### Figure X: [图表标题]

逐步拆解方法流程图：
1. **输入**：...
2. **步骤1**：...
3. **步骤2**：...
4. **输出**：...

物理/生物学含义：...

## 相关论文/引用线索

| 论文 | 关联 |
|------|------|
| Author et al., Year. Title | 为什么值得关注（一句话） |
```

## 输出规范

- 文件格式：Markdown
- 命名规则：`[方法名或关键词]_[第一作者姓]_[年份]_笔记.md`
  - 例：`SpaGCN_Hu_2021_笔记.md`
- 语言：中文为主，专业术语保留英文
- 长度：控制在1-2页（约800-1200字）
- "个人思考/评价"部分留空（用引用块标注），由用户自行补充

## 注意事项

- 精简优先：每个模块宁可少写，不要堆砌冗余信息
- 引用数如果搜索结果不确定（比如同名论文多篇），标注"约 X 次（待确认）"
- 图表分析聚焦方法流程图，不需要分析所有图表
- 相关论文从原文References中挑选，不额外搜索补充
